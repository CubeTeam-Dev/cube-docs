# Cube Bot ドキュメント 貢献ガイドライン

Cube Bot のドキュメントプロジェクトへ興味を持っていただき、ありがとうございます！
このプロジェクトは **オープンソース (OSS)** であり、コミュニティによる改善、誤字脱字の修正、および他言語への翻訳を心より歓迎しています。

---

## 🚀 貢献の始め方

ドキュメントの内容に誤りを見つけた場合や、新しい機能の説明を追加したい場合は、**プルリクエスト (Pull Request)** を通じて以下の手順で提案してください。

1.  このリポジトリをフォーク (Fork) します。
2.  新しい機能や修正のためのブランチを作成します。
3.  変更をコミットし、プッシュ (Push) します。
4.  プルリクエストを作成します。

💡 **ヒント**: 大規模な変更を予定している場合は、事前に Issue で提案をいただけるとスムーズです。

---

## 🌐 翻訳・言語の追加

Cube Bot は世界中での利用を目指しており、ドキュメントの多言語化を推進しています。

### 既存ページの翻訳・修正
1.  `src/content/docs/` 配下にある対象と言語のディレクトリを確認します。
2.  対応する `.mdx` ファイルを編集。あるいは、日本語版をベースに他言語ディレクトリへファイルをコピーして翻訳を開始してください。
3.  日本語版と構成を同期させることを推奨します。

### 新しい言語をサポートするフロー
新しい言語（例：フランス語 `fr`）を追加する場合は、以下の手順が必要です。

1.  **設定の更新**: `astro.config.mjs` の `starlight.locales` セクションに新しい言語定義を追加。
    ```js
    locales: {
      ja: { label: "日本語", lang: "ja" },
      fr: { label: "Français", lang: "fr" },
      // 新しい言語を追加
    },
    ```
2.  **ディレクトリ作成**: `src/content/docs/` 配下に新しい言語コードのディレクトリ名（例：`fr/`）を作成。
3.  **コンテンツの配置**: 翻訳されたドキュメントを配置。

## 📏 規約とルール

プロジェクトの品質と保守性を維持するため、以下のルールを遵守してください。

### ファイル命名規則
*   **ファイル名・ディレクトリ名**: すべて **小文字のケバブケース (kebab-case)** で統一してください。
    *   ✅ 正しい: `server-settings.mdx`, `manage-commands/`
    *   ❌ 誤り: `ServerSettings.mdx`, `manage_commands`
*   **拡張子**: すべてのドキュメントは `.mdx` 形式で作成してください。

### MDX 記述の注意点
MDX パーサーの仕様により、Starlight コンポーネントを使用する際は以下の点に注意してください。

*   **コンポーネント内の空行**: `<Tabs>`, `<TabItem>`, `<Aside>` などのコンポーネントタグの内側にコンテンツを書く場合、**タグの直後と直前に必ず空行** を入れてください。
*   **TabItem 内の見出し**: `<TabItem>` の中で Markdown の見出し (`###`) を使うとレンダリングが崩れることがあります。代わりに **太字 (`**使い方**`)** を使用してください。
*   **ステップ表示**: 順序付きリスト (`1. ...`) は、自動的にスタイリッシュな「ステップ表示」としてレンダリングされるカスタムスタイルが適用されています。

## 💻 ローカルでのプレビュー

貢献の前に、ローカル環境で表示を確認することを推奨します。

1.  **依存関係のインストール**: `pnpm install`
2.  **開発サーバーの起動**: `pnpm dev`
3.  **表示確認**: [http://localhost:4321](http://localhost:4321) でページをプレビュー。
